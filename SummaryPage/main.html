Einleitung

In der IT-Welt gibt es kaum jemanden, der noch nie eine Browser Extension verwendet hat. Das Projektteam hatte noch keine Erfahrung mit Chrome Extensions, aber ein starkes Interesse zu lernen, wie diese im Hintergrund funktionieren und implementiert werden

Projektidee

Jeder Student kennt das Problem, dass er am Laptop arbeiten möchte, aber Ablenkungen wie Youtube oder Instagram nur einen Klick entfernt sind. Aufbauend auf die Pomodoro Technik, nach der in 25 Minuten-Zyklen gelernt wird, bietet die Chrome-Extension Timer-Funktionalitäten. Nach abgelaufenen Timer werden dabei Rewards in Form von Tieren freigeschaltet, um eine zusätzliche Lern-Motivation bereitzustellen. Während Lernzyklen werden dabei benutzerdefinierte Seiten blockiert, falls man durch die Macht der Gewohnheit doch einmal auf Social-Media Seiten abdriftet.

Installationsanleitung

1. Verwaltung der Chrome Extensions aufrufen: chrome://extensions/

2. Auf "Entpackte Erweiterung laden" klicken und das Projekt Focus Zoo auswählen

3. Fertig! Jetzt kann die Extension jederzeit über das Puzzle-Symbol geöffnet werden

Implementierung

manifest.json

Eine Chrome Extension ist eine ganz normale Webseite. Der einzige Unterschied ist ein eigenes manifest File, in dem Einstellungen wie Name, App-Icon etc. festgelegt wird

Codebeispiel manifest.json 
            
{
  "name": "Focus Zoo",
  "version": "0.0.1",
  "description": "This extension helps you to stay focussed while building you own zoo",
  "icons": {
    "16": "images/camel16.png",
    "48": "images/camel48.png",
    "128": "images/camel128.png"
  },
  "manifest_version": 3,
  "action": {
    "default_popup": "pages/index/index.html",
    "default_title": "Focus Zoo"
  },
  "permissions": [
    "storage"
  ],
  "background": {
    "service_worker": "db/service_worker.js",
    "type": "module"
  },
  "content_scripts": [
    {
      "matches": [
        ""
      ],
      "js": [
        "pages/blocked_override/blocked_override.js"
      ]
    }
  ],
  "web_accessible_resources": [
    {
      "resources": [
        "pages/timer/clock.js",
        "db/database.js",
        "db/countdown.js",
        "db/websites.js"
      ],
      "matches": [
        ""
      ]
    }
  ]
}

Datenbanken

Damit die freigeschalteten Tiere und laufenden Timer immer gespeichert werden, muss auch eine Datenbank verwendet werden. Tabellen: - animals: Speichert freigeschaltete Tiere - countdowns: Speichert abgeschlossene Countdowns - websites: Speichert blockierte Seiten Für gespeicherte Daten innerhalb der Extension wird dabei eine IndexedDB verwendet. Um auf externen Seiten Zugriff auf die Datenbank zu ermöglichen, werden Tabellen wie websites im Chrome-Local-Storage gespeichert, weil die IndexedDB nur innerhalb der Extension verfügbar ist. Überschrift

Lessons learned

Arbeiten mit Chrome Extensions

Es war sehr überraschend, wie einfach Chrome Extensions erstellt werden können. Für jeden mit Interesse kann das Experimentieren mit Chrom Extensions auf jeden Fall empfohlen werden

Arbeiten als Team

Vorhandene GIT-Vorkenntnisse haben das gemeinsame arbeiten deutlich vereinfacht. Dabei wurde für jede Änderung ein eigener Merge Request angelegt, um eine gute Code-Qualität garantieren zu können

Vorgangsweise und Arbeitsteilung

Während Mathias Pichler schon früh mit dem Grunddesign und der Backendprogrammierung begann, übernahm Marielle Wildauer danach die Feingliederung des Inhaltes inklusive Design. Diese Vorgangsweise ermögliche es, kaum Konflikte im Code zu erzeugen. Dies führte zu einer schnellen Implementierung des Projektes. Außerdem wurde bereits zu Beginn in Notion ein einfaches Ticket-System aufgesetzt, womit die Arbeit strukturiert und übersichtlich dargestellt wurde.
